# Git Bloom

**Grow your garden with every commit.**

Git Bloom is an interactive web-based game where coding meets cultivation. Turn your repository's version control activity into a thriving, beautiful digital garden. 

* **Play the Game:** [Live Deployment on Vercel](https://git-bloom-sooty.vercel.app/)
* **How to Play:** [Game Rules & Instructions](https://git-bloom-sooty.vercel.app/how-to-play)

---

## Game Overview

In **Git Bloom**, your Git history serves as the nourishment for your environment. Every action you take—from committing code to merging branches—influences the growth, varieties, and layout of the plants in your digital ecosystem. It bridges the gap between software development tracking and relaxing simulation gameplay.

### Key Features
* **Dynamic Generation:** Watch your plants evolve based on real or simulated git command histories.
* **Garden Management:** Customize your patch, clear weeds, and experiment with different branch patterns to unlock rare flora.
* **Relaxing Aesthetic:** Enjoy a serene pixel art and ambient atmosphere designed to reduce coding burnout.

---

## How to Play

For a comprehensive breakdown of the mechanics, mechanics formulas, and plant behaviors, make sure to visit the [Official How to Play Guide](https://git-bloom-sooty.vercel.app/how-to-play). 

### The Core Loop:
1. **Commits = Seeds:** Regular pushes introduce new plant sprouts into your soil beds.
2. **Branches = Growth Paths:** Branching your code branches your garden pathways, allowing for unique visual structures.
3. **Merges = Cross-Pollination:** Combine branches to create beautiful hybrid flowers and unlock special achievements.

## Bug Reports & Feedback

We want to make Git Bloom as stable and polished as possible! Because this main game engine code is kept in a private repository, we track issues externally to protect our source files.

If you encounter any bugs, glitches, or visual issues, please visit our **[Public Bug Tracker](https://github.com/Sensiblemnd/git-bloom-feedback/issues)** to submit an issue. 

### How to report an issue:
1. Navigate to the **Issues** tab on our public feedback repo.

3. Describe the problem, list your browser type, and provide screenshots if possible.
